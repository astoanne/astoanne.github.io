rPCO 图解

rPCO 到底在干什么？

一句话：让主模型只负责"形状"，把每个簇的"高低"单独记下来再加回去。

1背景：数据里藏着"分组差"

真实数据常常分成几个"簇"（cluster）。同一个 x，不同簇的 y 整体水平就是不一样高。

类比：两个学校的考试。 A 校学生平均 85 分，B 校学生平均 75 分。
把"学习时长 → 成绩"画在图上，两校的斜率（关系形状）差不多， 但两校的整体水平差了 10 分。
图 1 ｜ 两个簇形状相似（斜率近），但整体高低差很大。

2没有 rPCO：模型被"拉扯"

FFSRQ 原版默认所有簇共享同一组基底系数。它会试图用一条线同时穿过两个簇 —— 结果是：

  • 线被拉到两簇中间，对 A 校太低、对 B 校太高；
  • 为了"凑"高度，模型把基底预算浪费在"猜平均水平"上，没法精细拟合形状；
  • 残差大，预测 MSE 高。
图 2 ｜ 一条妥协线让两簇都不准（红色箭头 = 残差）。

3加上 rPCO：拆成两步

residual-based Per-Cluster Offset =「基于残差的，每个簇一个的，截距修正」。

  1. 第一步：让 FFSRQ 主模型先按原方式拟合（学"形状"），得到每个样本的预测 ŷ。
  2. 第二步：对训练集每个簇 k，算出"残差的平均值" Δk = mean(y − ŷ)。
  3. 第三步：把 Δk 当作这个簇的"修正常数"存起来。
  4. 预测时：对一个属于簇 k 的测试样本，预测 = ŷ + Δk。
图 3 ｜ 形状只学一次，水平差由每个簇的常数 Δ 各自吸收。
类比延续： 老师先用整体规律（学习时长 → 分数变化）打分，然后给每个学校加上一个校准分：A 校 +5，B 校 −3。这样既保留了整体规律，又承认了校间差距。

4有什么提升

  • 主模型不再被"高低差"挤占基底预算，能专心学形状；
  • Δk 是用训练残差算的，不引入新的待优化参数，不会过拟合；
  • 只在 clust.num > 1 时启用；只通过 FFSRQ_CLUSTER_INTERCEPT=1 控制 —— 完全向后兼容，可一键开关；
  • 实验上，模拟数据（50 次重复）+ 4 个 main 真实数据集（Air / Ocean / Population / Stock）+ 1 个 backup（Energy）下，FFSRQ 在 13 个对比方法中拿到 3× rank-1 + 1× tied rank-1 + 1× rank-2（0.9%），backup 上 rank 1。

五个真实数据集（4 main + 1 backup）

🌊 Ocean（夏威夷海洋观测）
每个样本是一条深度剖面（按深度连续测量）。
输入：
  log(temp) — 海水温度（℃）
  chlo — 叶绿素 a 浓度（浮游植物多少的代用量，mg/m³）
输出：
  log(sal) — 海水盐度（PSU）
用温度和叶绿素剖面预测盐度剖面。
🌫️ Air（空气质量监测）
意大利某城市的空气质量传感器小时级时序曲线。
输入：
  log(temperature+1) — 气温（℃）
  log(humidity) — 相对湿度（%）
  NOx — 氮氧化物浓度（μg/m³）
输出：
  NO2 — 二氧化氮浓度（μg/m³）
用气象 + NOx 的曲线预测 NO₂ 的曲线。
⛽ Energy†（美国汽油零售价）— BACKUP
1994-2024 美国各区域每周零售汽油价格，按年组织成 52 周曲线（仅 n=30 年）。
输入：
  OT — 总体均价
  EastCoast — 东海岸均价
  NewEng — 新英格兰均价
输出：
  WestCoast — 西海岸均价
用东部各区域的年内周价曲线预测西海岸 —— FFSRQ rank 1，但 n=30 偏小，作为 backup 验证。
🌍 Population（World Bank 人口指标）
184 个国家 / 地区 × 1961-2015（55 年）World Bank 人口数据。
输入：
  birth_rate — 粗出生率
  pop_growth — 总人口年增长率
  rural_growth — 农村人口增长率
输出：
  death_rate — 粗死亡率
用出生率/人口增长/农村增长曲线预测死亡率曲线（80/20，FFSRQ rank 1，margin 0.6%）。
📈 Stock（股价）
四只大科技股的日价格曲线，共 457 个月，每月 22 个交易日。每月内归一到 [0,1]。
输入：
  AAPL — 苹果，GOOGL — Alphabet/谷歌，NVDA — 英伟达
输出：
  MSFT — 微软
用三只科技股的月内走势预测微软的月内走势。

为什么有些变量套了 log？ 海洋的温度/盐度、空气的温度/湿度这类正实数取值跨度大、分布右偏，套 log 能压住尾部、让方差更稳，关系也更接近线性 —— 是函数型回归里很常规的预处理。
为什么是 log(temperature+1) 而不是 log(temperature)？ 因为气温会贴近 0（℃），log(0) = −∞；加 1 避免奇点。
为什么 chlo、NO2、NOx 没 log？ 它们的取值范围本来就比较窄、分布没那么偏，套 log 反而会扭曲；股价已经做了 0–1 归一化，也不需要 log。

5这种做法常见吗？相关文献

在函数型回归里，"加随机截距函数 / 逐组截距"是很常见的思路：

  • Morris & Carroll (2006), JRSS-B — wavelet-based functional mixed model，逐 subject 截距函数；
  • Scheipl, Staicu & Greven (2015), JCGS — pffr / refund 包里就支持 random intercept 项；
  • Goldsmith, Zipunnikov & Schrack (2015), Biometrics — multilevel function-on-scalar regression。

6代码层面到底改了什么

在 Anne_rayleigh_quotient0924.R 的 mainFFSRQ() 里增加了一个可选模块，对原损失函数、β 估计与主优化流程零修改。两处改动如下。

① 开关声明

# per-cluster intercept switch (computed at end of mainFFSRQ)
use.cluster.intercept <- identical(Sys.getenv("FFSRQ_CLUSTER_INTERCEPT", ""), "1")

② 训练末尾的残差修正块

# per-cluster intercept: membership-weighted residual mean per cluster,
# smoothed on the y-basis with eta penalty, added back to Y.hat
if (use.cluster.intercept && clust.num > 1) {
  residuals.train <- output.ori - Y.hat                         # ntrain x len.t
  weights_t      <- t(active.k)                                 # K x ntrain
  weight_sums    <- rowSums(weights_t) + 1e-12
  cluster.resid.mean <- sweep((weights_t %*% residuals.train), 1, weight_sums, "/")
  w.0.cluster <- matrix(0, nrow = length(t), ncol = clust.num)
  smoother    <- pinv(int.bly + eta * int.bly2)
  for (m in 1:clust.num) {
    int_mean_m <- numeric(M)
    for (i in 1:M) {
      int_mean_m[i] <- sum(cluster.resid.mean[m, ] * bl.y.des[, i] * (1/(len.t-1)))
    }
    w.0.cluster[, m] <- as.numeric(bl.y.des %*% smoother %*% as.matrix(int_mean_m, ncol = 1))
  }
  Y.hat <- Y.hat + active.k %*% t(w.0.cluster)
}

预测端只需把 w.0.cluster 存进模型对象，再在 predict.mainFFSRQ() 里用测试样本的 membership 把对应簇的修正项加回 Y.hat 即可，不展开。

7论文 method section 草稿

Residual-based Per-Cluster Offset (rPCO).

To capture cluster-specific level shifts in $y(t)$ that cannot be absorbed by the consequent parameters, we introduce a post-hoc correction term, the residual-based per-cluster offset (rPCO). After the antecedent and consequent parameters have been estimated, let $\hat{y}_i(t)$ denote the in-sample prediction of $y_i(t)$ and $\mu_k(x_i(s))$ the membership of sample $i$ in cluster $k$. For each $k = 1, \ldots, K$, the per-cluster residual centre is

$$ \hat{\Delta}_k(t) \;=\; \frac{\displaystyle\sum_{i=1}^{N}\mu_k(x_i(s))\bigl(y_i(t)-\hat{y}_i(t)\bigr)}{\displaystyle\sum_{i=1}^{N}\mu_k(x_i(s)) + \varepsilon}, $$

with $\varepsilon > 0$ a small constant for numerical stability. Each $\hat{\Delta}_k(t)$ is then smoothed by projection onto the output B-spline basis $\{B^{Y}_q(t)\}_{q=1}^{Q}$ under the same roughness penalty $\eta$ used in the main estimator, yielding $\tilde{\Delta}_k(t)$. The final prediction is

$$ \tilde{y}(t) \;=\; \hat{y}(t) \;+\; \sum_{k=1}^{K}\beta_k\bigl(x(s)\bigr)\,\tilde{\Delta}_k(t), $$

where $\beta_k(x(s))$ is the normalised functional membership of $x(s)$ in cluster $k$. rPCO introduces no additional hyperparameter and is only activated when $K > 1$.

\subsection{Residual-based Per-Cluster Offset (rPCO)}

To capture cluster-specific level shifts in $y(t)$ that cannot be absorbed by the consequent parameters, we introduce a post-hoc correction term, the \emph{residual-based per-cluster offset} (rPCO). After the antecedent and consequent parameters have been estimated, let $\hat{y}_i(t)$ denote the in-sample prediction of $y_i(t)$ and $\mu_k(x_i(s))$ the membership of sample $i$ in cluster $k$. For each $k = 1,\ldots,K$, the per-cluster residual centre is
\begin{equation}
\hat{\Delta}_k(t) = \frac{\sum_{i=1}^{N}\mu_k(x_i(s))\bigl(y_i(t)-\hat{y}_i(t)\bigr)}{\sum_{i=1}^{N}\mu_k(x_i(s)) + \varepsilon},
\end{equation}
with $\varepsilon > 0$ a small constant for numerical stability. Each $\hat{\Delta}_k(t)$ is then smoothed by projection onto the output B-spline basis $\{B^{Y}_q(t)\}_{q=1}^{Q}$ under the same roughness penalty $\eta$ used in the main estimator, yielding $\tilde{\Delta}_k(t)$. The final prediction is
\begin{equation}
\tilde{y}(t) = \hat{y}(t) + \sum_{k=1}^{K}\beta_k\bigl(x(s)\bigr)\,\tilde{\Delta}_k(t),
\end{equation}
where $\beta_k(x(s))$ is the normalised functional membership of $x(s)$ in cluster $k$. rPCO introduces no additional hyperparameter and is only activated when $K > 1$.
