📤 translationchina.com 上传方法变了

迁移到云服务器之后,用 SFTP 替代之前的 FTP

简单说:网站现在在新的云服务器上了,**之前用 FileZilla 连的 FTP 失效了**。 下面教你换成 SFTP,基本一样,只是协议和地址变了。

🔒 主机 IP 和密码不能在这个公开页面里写。Anne 会通过微信单聊另外发给你。 你拿到后填在 FileZilla 里就行。

🛠️ FileZilla 设置步骤

1 打开 FileZilla → 站点管理器
顶部菜单点 文件 → 站点管理器(快捷键 Ctrl+S)。 左侧找到原来的"translationchina"那个站点 — 我们要在它旁边新建一个。
2 新建站点
点 新站点,起个名字叫 translationchina 云服务器(或者你顺手的名字)。
3 填连接信息
右侧"常规"标签里填:
协议SFTP - SSH File Transfer Protocol
主机(Anne 微信发的 IP)
端口22000
登录类型正常
用户root
密码(Anne 微信发的密码)
注意端口是 22000,不是默认的 22。填错会连不上。
4 设默认远程目录(可选)
点 高级 标签 → "默认远程目录" 填: 
/var/www/translationchina/html/
这样连上就直接在网站根目录,不用手动找。
5 连接
点 连接 按钮。第一次连接会问"未知主机密钥",勾选"始终信任",然后确定。
✅ 看到右侧出现一堆文件 + 文件夹(1 / 1207 / notary / airports / index.html ...)就成功了。

📁 跟之前比有什么不同?

  • 整体目录结构一样。你之前传文件到哪个文件夹,现在还是传到那个文件夹。
  • 看到的 notary/、airports/ 是新加的目录,别动它们。
  • 看到的 index.html、enindex.htm、1120备份/ 等等都是原来你熟悉的文件,完全没动。
  • 大部分操作跟之前一样:左边电脑,右边服务器,拖拽传。

🧪 怎么确认现在网站真的走云服务器了?

给 Anne 看的验证方法(她可能想自己确认):

方法 1:浏览器 DevTools

  1. 打开 https://www.translationchina.com/
  2. 按 F12 打开开发者工具 → Network 标签
  3. 刷新页面(Ctrl+F5 强刷)
  4. 点列表里第一条(通常是主页文档),看右侧 Headers:
    • Remote Address 应该是 211.149.228.244:443(云服务器)
    • Server header 应该是 nginx/1.18.0 (Ubuntu)(老服务器是 wts/1.x.x)

方法 2:命令行(更快)

curl -sI https://www.translationchina.com/ | head -5

看到的应该是:

HTTP/1.1 200 OK
Server: nginx/1.18.0 (Ubuntu)   ← 新服务器
...

如果还是 Server: wts/1.x.x,那是 DNS 缓存,等 10-30 分钟自然就刷过来。

方法 3:DNS 查询

nslookup www.translationchina.com 8.8.8.8

地址应该是 211.149.228.244。如果是 211.149.244.115 或 211.149.229.49 等旧 IP,那是 DNS 还没传播完,等。

方法 4:用无痕窗口确认不是浏览器缓存

打开 Chrome 无痕窗口(Ctrl+Shift+N)→ 访问 https://www.translationchina.com/ → F12 看 Server header。无痕窗口没缓存,绝对反映真实状态。

❓ 万一连不上 FileZilla 怎么办

  1. "连接超时" / "无法连接" → 检查协议是不是 SFTP(不是 FTP)、端口是不是 22000(不是 21 也不是 22)
  2. "密码错误" → 让 Anne 重新发一次密码
  3. "主机密钥已更改" → 在站点管理器里删了重新加这个站点
  4. 仍然不行 → 截图错误信息发给 Anne
